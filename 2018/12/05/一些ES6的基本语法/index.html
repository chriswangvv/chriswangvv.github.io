块级作用域

const

  1. 一旦声明，必须立即初始化；不提升；暂时性死区；不可重复声明

  2. 本质： 变量指向的内存地址所保存的数据不能改动。

    所以，复合类型的数据（对象和数组），变量指向的内存地址保存的只是一个指向实际数据的指针，const只能保证这个指针总是指向另一个固定的地址（所以，const arr[]依然能往里push数据）

  3. freeze **

    const foo = Object.freeze({});
foo.prop = 123
console.log(foo.prop)

var constantize = (obj) => {
    Object.freeze(obj);
    Object.keys(obj).forEach((key, i) => {
        if (typeof obj[key] === 'object') {
            constantize(obj[key])
        }
    })
}

变量的解构赋值

数组的解构赋值

  1. 模式匹配

    如果解构不成功，变量的值就等于undefined

    例子：

    let [foo, [[bar], baz]] = [1, [[2], 3]]
foo === 1
bar === 2
baz === 3

let [ , , third] = ["foo", "bar", "baz"];
third // "baz"

let [head, ...tail] = [1, 2, 3, 4];
head // 1
tail // [2, 3, 4]

let [x, y, ...z] = ['a'];
x // "a"
y // undefined
z // []

  1. 不完全解构

    例子：

    let [x, y] = [1, 2, 3];
x // 1
y // 2

let [a, [b], d] = [1, [2, 3], 4];
a // 1
b // 2
d // 4

    如果等号的右边不是数组（或者严格地说，不是可遍历的结构，参见《Iterator》一章），那么将会报错。

    报错例子：

    let [foo] = 1;
let [foo] = false;
let [foo] = NaN;
let [foo] = undefined;
let [foo] = null;
let [foo] = {};

    对于Set结构，也可以使用数组的解构赋值

    let [x, y, z] = new Set(['a', 'b', 'c'])

    实际上，只要某种数据接口具有Iterator接口，都可以采用数组形式的解构赋值

    function* fibs() {
    let a = 0;
    let b = 1;
    while (true) {
        yield a;
        [a, b] = [b, a + b]
    }
}

let [first, second, third, fourth, fifth, sixth] = fibs();
sixth === 5

    fibs是一个Generator函数，原生具有Iterator接口，解构赋值会依次从这个接口获取值

  2. 默认值

    解构赋值允许指定默认值

    例子：

    let [foo = true] = []
foo ===> true
let [x, y = 'b'] = ['a']
let [x, y = 'b'] = ['a', undefined]

    注意，ES6 内部使用严格相等运算符（===），判断一个位置是否有值。所以，只有当一个数组成员严格等于undefined，默认值才会生效
